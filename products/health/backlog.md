# Keel — Backlog

Post-MVP items, prioritized. All pre-launch items are tracked in the GitHub v1.0 milestone.

---

## High Priority (next after launch)

**#7 — Push notifications and reminders**
Zero re-engagement hooks. No nudges for logging meals, completing workouts, or ending fasts. High retention impact.

**#13 — Food log editing**
Currently can only delete log entries via long press. Editing a logged item (quantity, meal type) is a basic expectation. Competitor gap.

**#19 — Recent/favorite foods in food search**
Cold start every search — no recents, no favorites. High-frequency friction. Every competitor has this.

**#22 — Workout finish validation**
Workout can be marked complete with zero sets logged. No guard against accidental finish.

---

## Medium Priority

**#23 — HealthKit / Health Connect integration**
Passive data (steps, heart rate, sleep) would significantly improve the health picture without requiring manual input. Gated on Apple / Google approval processes.

**#20 — Set completion undo**
Easy to accidentally mark the wrong set complete. No undo. Low effort, decent polish value.

**#16 — Fasting timer background notification**
Timer doesn't notify when the fast completes if app is in background. Core feature of any fasting timer.

**#5 — Privacy dashboard completion**
Gamification and fasting tables missing from the data inventory in the privacy dashboard. Important for the privacy story long-term.

**#6 — Data export (GDPR)**
Currently shows "coming soon." Required for GDPR right to portability. Not launch-blocking for initial US/Canada launch but needed before any EU marketing.

---

## Low Priority

**#17 — Superset/circuit training support**
Competitors (Strong, Hevy) support supersets. Niche but vocal user segment.

**#14 — Workout notes**
Notes on sets and sessions. Nice-to-have for serious lifters.

**#15 — Exercise images/videos**
Text-only instructions currently. Images would reduce support burden and improve onboarding for new lifters.

**#21 — Loading skeleton screens**
Blank screens while data loads. Polish item — low user impact given local-first architecture (loads are fast).
